# RGB-D object segmentation and pose estimation

This repository implements a small **sim-to-real style** pipeline: a **Mini-UNet** segments five YCB-style objects in RGB images using **instance masks** from a simulated camera, then **depth + masks** are used to build partial point clouds and align **mesh models** with **ICP** (via Trimesh) to estimate each object’s **6D pose**.

```mermaid
flowchart LR
  A[PyBullet simulation] --> B[RGB + instance masks]
  B --> C[Train Mini-UNet]
  C --> D[Predicted masks]
  D --> E[Depth unprojection + ICP]
  E --> F[Per-object 4x4 poses]
```

## What’s inside

| Piece | Role |
|--------|------|
| `gen_dataset.py` | Renders **training** RGB and ground-truth masks in PyBullet (YCB subset meshes). |
| `segmentation.py` | Trains `MiniUNet` on `dataset/train/`, validates, writes `checkpoint.pth.tar`, saves predictions to `dataset/*/pred/`. |
| `icp.py` | Loads depth, predicted (and optionally GT) masks, runs **pose estimation + optional PLY export**. |
| `evaluate_icp.py` | **Validation** only: compares estimated poses to `dataset/val/gt_pose/` using mean closest-point distance on mesh samples. |

Object models live under `YCB_subsubset/` (sugar box, soup can, tuna can, banana, bowl). Class IDs `1–5` match that order.

## Requirements

- Python **3.10+** recommended (tested with 3.13).
- CUDA optional; training and inference run on CPU if needed (slower).

Install dependencies:

```bash
pip install -r requirements.txt
```

## Quick start

### 1. Generate training data (optional)

If you want to re-render the **training** set:

```bash
python gen_dataset.py
```

This uses PyBullet with a GUI window. It writes under `dataset/train/rgb/` and `dataset/train/gt/`. The included repo already contains a full training split (~300 RGB frames).

### 2. Train the segmentation model

```bash
python segmentation.py
```

- Reads `dataset/train/`, `dataset/val/`, `dataset/test/`.
- Saves the best checkpoint to `checkpoint.pth.tar` (training resumes if the file exists).
- Writes mask visualizations to `dataset/val/pred/` and `dataset/test/pred/`.
- Saves `learning_curve.png`.

> **Note:** `segmentation_local.py` is a lighter variant (e.g. fewer epochs / different dataloader settings) kept for quick local experiments. The main entry point is `segmentation.py`.

### 3. Estimate object poses (RGB-D + masks)

After segmentation has produced `*_pred.png` masks:

**Validation** (ground-truth and predicted masks; also exports PLY and `gtmask` poses for comparison):

```bash
python icp.py --val
```

**Test** (predicted masks only):

```bash
python icp.py --test
```

Poses are saved as NumPy arrays: `dataset/<split>/pred_pose/predmask/<scene_id>_<obj_id>.npy` (and `gtmask/` on validation when using `--val`).

### 4. Evaluate on validation (optional)

Requires ground-truth poses in `dataset/val/gt_pose/`:

```bash
python evaluate_icp.py --predmask
python evaluate_icp.py --gtmask
```

Reports per-object **average closest-point distance** between mesh points transformed by predicted vs. ground-truth poses.

## Method summary

1. **Segmentation:** Encoder–decoder CNN (`MiniUNet`) with cross-entropy loss; mIoU tracked per epoch. Input RGB is normalized using dataset statistics; training uses flips and small rotations.
2. **Pose:** For each object ID, depth is masked, unprojected to the camera, transformed to the world using the stored **view matrix**, then aligned to **points sampled from the mesh** using Procrustes initialization + **ICP** (`trimesh.registration`).
